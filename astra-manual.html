Section 01

5-Minute Quick Start

Open ASTRA. Do these 5 things. You now know 80% of the app.

THE ONE THING TO REMEMBER

Cmd+K opens the Command Palette. It does everything. Search, create tasks, switch sections, run commands. If you forget every other shortcut, remember this one.

Step 1: Explore the 9 Sections

The left sidebar has 9 sections. Click each one to see what it does. Or use keyboard shortcuts:

Cmd+1
Content
Quick notes, ideas, snippets
Cmd+2
Tasks
List view + Kanban board
Cmd+3
Calendar
Month, week, year heatmap
Cmd+4
Writer
Long-form documents
Cmd+5
Living Docs
Specs with sections
Cmd+6
Links
Bookmarks + categories
Cmd+7
Whiteboard
Visual canvas + nodes
Cmd+8
Pipeline
Text processing tools
Cmd+9
Repos
Track your code repos

Step 2: Create Your First Task

Press Cmd+K
The Command Palette opens. This is your universal search + action bar.
Type task: Build my first project
The task: prefix creates a task instantly. Hit Enter.
Go to Tasks (Cmd+2)
Your task appears. Click it to set priority, due date, description, subtasks.

Step 3: Save a Quick Note

Press Cmd+K again
Type note: This is my first idea. Hit Enter.
Check Content (Cmd+1)
Your note is saved as a content card. You can tag it, pin it, search it.

Step 4: Open a Project

In the sidebar, click any project name (e.g., "Claude Code" or "Saturno Bonus"). A project panel slides open from the right with:

  • Instructions — free-text project notes (editable)
  • Context — Claude.ai-style quick actions (Plan & Work, Research, Create Context, Create Prompt)
  • Living Docs — structured specs for the project
  • Knowledge Base — facts, rules, references
  • Tasks — filtered to this project only
  • Links — project-specific bookmarks

Step 5: Try the Writer

Press Cmd+4
Opens the Writing Hub. Click "+ New Doc" to start a document.
Press Cmd+Shift+W
Zen Writer mode. Full-screen, distraction-free writing. Press Escape to exit.
CONGRATULATIONS

You now know the 5 core moves: navigate sections, create tasks, save notes, open projects, and write documents. Everything else in this manual builds on these foundations.

Section 02

The Layout (3 Panels)

ASTRA has 3 panels. Left sidebar, center workspace, right context. That's it.

+------------------------------------------+ | HEADER [logo] search [mode] [stats] | +------+-------------------------+---------+ | | | | | SIDE | MAIN WORKSPACE | CONTEXT | | BAR | | PANEL | | | (content, tasks, | | | sections| calendar, writer, | timer | | projects| whiteboard, etc.) | focus | | folders | | notes | | | | | +------+-------------------------+---------+

Left Sidebar

Contains 3 things stacked vertically:

  1. Section icons — Click to switch between the 9 sections (Content, Tasks, Calendar, Writer, Living Docs, Links, Whiteboard, Pipeline, Repos)
  2. Projects — Your projects. Click one to open the project panel on the right.
  3. Folders — Nested folder tree that mirrors your filesystem structure.

Collapse the sidebar with Cmd+B or click the << chevron. In collapsed mode, only icons show.

Center Workspace

This is where you work. It shows whichever section you selected. Each section has its own toolbar at the top with search, filters, sort, and action buttons.

Right Context Panel

Shows context-sensitive information. Toggle it by switching view modes (top-right dropdown):

  • Focus — Workspace only, no context panel
  • Default — Workspace + sidebar
  • Extended — All 3 panels visible (sidebar + workspace + context)
TIP

The project panel and context panel share the same space on the right. Opening a project closes the context panel, and vice versa.

Section 04

Tasks & Board

Two views of the same data: a sortable list and a Kanban board. Drag between columns.

Two Views

L
List View

Rows with title, priority color bar, due date, project badge. Click a row to open the detail modal. Sortable by priority, due date, newest, alphabetical.

B
Board View

Kanban columns: Todo, In Progress, Review, Done. Drag cards between columns. Cards show priority, due date, subtask progress, description preview.

Task Properties

Click any task to open its detail modal. You can set:

  • Priority: P0 P1 P2 P3 — click the priority badge on board cards to cycle
  • Due date: Manual date picker + quick buttons (Today, Tomorrow, +1 Week, +1 Month)
  • Description: Rich text body for details
  • Subtasks: Add checkboxes with "+ Subtask" button. Progress bar shows completion.
  • Recurrence: Daily, weekday, weekly, biweekly, monthly. Auto-creates next task on complete.
  • Project: Assign to a project. Task appears in that project's task list.
  • Estimate: Time estimate (e.g., "30m", "2h"). Shows in cyan on board cards.
  • Blocked by: Dependency field. Shows "B" badge when blocked.
  • Timer: Start/Stop timer button. Tracks actual time spent.
  • Focus: 25-minute Pomodoro timer that also tracks time.

Bulk Operations

In list view, check multiple tasks. A toolbar appears letting you set status, priority, or delete in bulk.

Quick Shortcuts

Content Vault

Your catch-all for ideas, notes, quotes, snippets. Think of it as a smart notebook.

How Content Works

Content items are cards with a body, optional title, tags, and creation date. They're the simplest thing in ASTRA — just text you want to save.

Creating Content

  • Cmd+K then note: your text here — instant save
  • Cmd+N (Quick Capture popup) — write and tag
  • "+ Add" button in the Content toolbar
  • Quick Note widget in the Context Panel (right side)

Features At a Glance

Tags & Filtering

Add tags to content. Click a tag pill to filter by it. Rename or delete tags across all items from the toolbar.

Pin & Star

Pin items to sort them to the top (green border). Star items for favorites (yellow indicator).

Sort & Search

Sort by newest, oldest, alphabetical, word count, pinned. Search highlights matches in green.

Actions

Double-click to edit inline. "Doc" converts to a Writer document. "Pipe" sends to Pipeline. Bulk select + delete/tag.

UNDO

Accidentally deleted content? Press Cmd+Z immediately to undo the last delete.

Section 06

Calendar

See your tasks on a timeline. Month view, week view, year heatmap, or agenda list.

4 Views

Month View

Classic calendar grid. Tasks appear as colored pills on their due dates. Click a day to create a task on that date. Drag tasks between days.

Week View

7-day focused view. Day headers show task counts. Easier for short-term planning.

Year Heatmap

GitHub-style contribution heatmap. Shows activity intensity for every day of the year. Green = more activity.

Agenda

Flat list of all upcoming tasks sorted by date. No grid — just the raw schedule.

Calendar Features

  • Drag and drop: Move tasks to different dates by dragging pills
  • Click to create: Click any day to create a task on that date
  • Day detail popup: Click a day number to see all tasks for that day
  • Export: Export as .ics file (importable into Google Calendar)
  • Import: Import .ics files from Google Calendar or other apps
  • Overdue + Upcoming: Lists below the calendar show overdue and upcoming tasks
TIP

Task pills are color-coded by priority. Red = P0, orange = P1, blue = P2, gray = P3. You can see at a glance which days have urgent tasks.

Section 07

Writing Hub

Long-form writing with versions, word goals, and a distraction-free Zen mode.

How It Works

Writer is for documents that grow over time — chapters, blog posts, scripts. Each document has a full rich-text editor with a version history.

Core Features

  • Rich editor: The center panel becomes a text editor. Just start typing.
  • Versions: Every version is saved. Click "Versions" to see history with word count diffs. Preview before restoring.
  • Word goal: Set a target word count. A progress bar tracks how close you are.
  • Auto-save: Saves a version every 10 minutes of editing automatically.
  • Find & Replace: Cmd+F or Cmd+H inside the writer.
  • TOC Generator: Click "TOC" to generate a Table of Contents from your headings.
  • Duplicate: Clone a document to start a new version from it.
  • Export: Markdown, HTML, or plain text.
  • To Content: "C+" button converts the doc to a content item.

Zen Writer

Press Cmd+Shift+W from anywhere. The entire screen becomes a focused writing environment — just you and the text. Press Escape to exit.

Section 08

Living Docs (Specs)

Structured documents with collapsible sections. Perfect for specs, plans, and technical documentation.

How They're Different From Writer Docs

Writer = one long document (like a Word file). Living Docs = a document split into named, collapsible sections with icons and colors (like a wiki page). Use Living Docs for reference material that stays organized; use Writer for flowing text you write top to bottom.

Working with Living Docs

  • Create: Click "+ New Spec" in the toolbar or go to a project's "Living Docs" tab
  • Sections: Each doc has multiple sections. Click "+ Section" to add. Click the collapse arrow to fold/unfold.
  • Icons & Colors: Click the icon next to a section title to cycle through 20 icons and 10 colors. Makes scanning easier.
  • Drag to reorder: Drag the handle next to a section title to rearrange sections.
  • Per-project: Assign a living doc to a project. It shows up in that project's panel.
  • Global docs: Docs without a project assignment appear everywhere.
  • Word count: Each section header shows its word count.
  • Copy section: "C" button copies a section as markdown.
  • Duplicate spec: Clone an entire living doc.
  • Export: "All Specs (MD)" or "All Specs (HTML)" from the export dropdown.
TIP

Living Docs support [[wiki-links]]. Type [[task title]] or [[doc name]] and ASTRA creates navigable links between items. View all backlinks with the "wiki stats" command in Cmd+K.

Section 16

Your Data (Storage & Export)

Where your data lives, how to back it up, and how to move it.

Where Data Lives

ASTRA stores everything in your browser's localStorage. This means:

  • Your data stays on YOUR computer. Nobody else has it.
  • It works completely offline. No internet needed.
  • Clearing browser data deletes your ASTRA data. Don't do this without backing up.
  • Different browsers have separate data (Safari and Chrome won't share).
IMPORTANT

Back up regularly. Your data is in localStorage. If you clear browser data or switch browsers, it's gone. Use the export features below.

How to Back Up

JSON Export (Full Backup)
Cmd+K → type "export" → select "Export Summary." This copies your full workspace as markdown. For JSON: open browser console and type localStorage.getItem('saturno-hub').
Section-Specific Exports
Tasks: "Export" button in toolbar (markdown). Calendar: "Export ICS" (importable). Writer: Export each doc as MD/HTML/TXT. Living Docs: "All Specs (MD)" or "All Specs (HTML)". Links: "Export All" as markdown.
Cloud Sync (Optional)
Press Cmd+S to sync to Vercel Blob storage (requires BLOB_READ_WRITE_TOKEN in Vercel env). This lets you access data from another device.

Import Options

  • ICS calendar files: Import from Google Calendar or Apple Calendar
  • Content batch import: Paste multiple items separated by ---
  • Links batch import: Paste multiple URLs, one per line
  • Whiteboard JSON: Import previously exported whiteboard data
